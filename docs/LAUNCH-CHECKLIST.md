# PlayPlayGG 上线前 Checklist

## 🔴 必须完成（阻塞上线）

### 部署
- [ ] Railway / VPS 上部署成功，`/health` 返回 200
- [ ] Webhook 模式验证通过（`WEBHOOK_URL` + `BOT_TOKEN` 配置正确）
- [ ] Redis 连接正常（`REDIS_URL` 配置后语言设置能持久化）
- [ ] Bot 的 Telegram webhook 设置正确（`getWebhookInfo` 无 pending_update_count 积压）

### E2E 手动测试（至少跑一遍）
- [ ] **5 人局**：开局 → 发言 → 讨论 → 投票 → 淘汰 → 平民胜 / 卧底胜
- [ ] **7 人局**（2 卧底）：验证多卧底分配和胜负判定
- [ ] **平票场景**：两人同票 → 无人淘汰 → 下一轮
- [ ] **无人投票**：20 秒超时 → 本局结束
- [ ] **白板局**：验证 10% 卧底白板和 5% 平民白板
- [ ] **有人没开私聊**：Bot 发词失败 → 不阻塞游戏，群里正常继续
- [ ] **有人中途不说话**：25 秒超时自动跳过，不卡住
- [ ] **结束游戏按钮**：投票阶段点「🛑 结束游戏」→ 游戏正常终止
- [ ] **多房间并行**：同一群同时开 2 局，互不干扰
- [ ] **语言切换**：`/playgg` → 换语言 → 文案全部跟随切换
- [ ] **私聊 `/start`**：返回欢迎消息 + 语言切换按钮
- [ ] **回跳链接**：私聊领词后的「返回群组」链接可以点击跳回

### 安全
- [ ] `.env` 里的 `BOT_TOKEN` 不要提交到 Git
- [ ] `dist/` 和 `node_modules/` 在 `.gitignore` 中

## 🟡 强烈建议（上线前 1 周内）

### 运营准备
- [ ] Bot 头像：设计一个有辨识度的 avatar（建议：🎭 面具 + 简洁配色）
- [ ] Bot 描述：在 @BotFather 设置 `/setdescription` 和 `/setabouttext`
- [ ] Bot 命令列表：`/setcommands` → `playgg - 打开游戏菜单`
- [ ] 准备 3 张游戏截图（报名 → 发言 → 投票结算）
- [ ] 准备 1 条推广短文案（50 字以内，适合转发）
- [ ] 准备 1 条群主邀请文案（说明拉 Bot 进群的好处）

### 种子测试
- [ ] 找 3-5 个真实群（朋友群、兴趣群）内测
- [ ] 记录：开局成功率、平均局时、用户吐槽点
- [ ] 至少跑 20 局，收集词库反馈：太简单 / 太难 / 没人懂的词

### 监控
- [ ] 进程崩溃自动重启（Railway 自带 / PM2 / systemd）
- [ ] 基础日志可查（Railway logs / `console.error` 输出有时间戳）

## 🟢 上线后持续做

### 词库迭代
- [ ] 建立词库淘汰清单：连续 3 局被秒猜的词 → 降级或移除
- [ ] 收集用户建议词对（可以做 `/suggest` 命令）
- [ ] 每周审阅一次词库，保持新鲜感

### 增长
- [ ] 游戏结束后的 growth 导流按钮文案定期更新
- [ ] 跟踪每日新增群数、每日活跃群数、每日开局数
- [ ] 考虑加「分享战绩图」功能，利用社交传播

### 稳定性
- [ ] Redis 房间持久化（进程重启不丢局）
- [ ] roomQuota 的 Redis 原子操作（Lua 脚本替代 get+incr）
- [ ] 结构化日志（pino / winston）
- [ ] 基础 Prometheus metrics（并发房间数、消息延迟）

## 推广节奏建议

**Week 1（灰度）**：3-5 个种子群，每天 1-2 局，收集 bug 和体验反馈
**Week 2（扩量）**：20-30 个群，修完种子期 bug，词库第一轮淘汰
**Week 3（开推）**：发推广文案，配截图，引导群主拉 Bot
**Week 4+（运营）**：每周更新词库，每两周加一个新游戏模块

核心指标：
- 开局成功率 > 80%（报名够人）
- 平均局时 < 10 分钟
- 次日留存群 > 30%
